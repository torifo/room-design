## 2. MVP 要件定義書
GitHub Pages等にそのままデプロイして動く、サーバーレス（LocalStorage完結）の要件定義です。
### 【1】システム概要
 * **プロダクト名**: （選定したサービス名）
 * **提供形態**: Webアプリ（PWA対応）
 * **デプロイ環境**: GitHub Pages または VPS（静的ファイル配信）
 * **コア価値**: 1つの部屋のサイズを指定し、実寸比率の家具（本棚、つくえ、Yogibo等）を2D Canvas上で直感的に配置・シミュレーションできる。(複数部屋を作成したら部屋間で物をドラックアンドドロップで移動できる)
### 【2】機能要件（MVP）
 1. **部屋（ベース）設定機能**
   * 部屋名（例: 和室、寝室）の入力。
   * 部屋の形（長方形のみでスタート）の選択。
   * 実際の縦・横サイズ（cm単位）の入力。
 2. **家具配置（Canvas）機能**
   * 家具の追加（本棚、つくえ、折りたたみ机、テレビ台、クッション、Yogibo）。
   * 家具の形状表現：本棚・机＝四角（Rect）、クッション・Yogibo＝円・楕円（Circle/Ellipse）。
   * ドラッグ＆ドロップによる移動。
   * ツマミ（Transformer）によるリサイズ・回転。
   * 家具の削除（選択して削除ボタン、またはDeleteキー）。
 3. **データ保存機能**
   * 現在の配置状態を LocalStorage に自動/手動保存。
 4. **「β版・今後の予定」明記（仕様）**
   * ヘッダーに「Beta」バッジを表示。
   * サイドバー等に「3D表示」「複数部屋の結合」「家具デザインの追加」などをグレーアウト or 「Coming Soon」として配置し、今後の拡張性をアピール。
### 【3】非機能要件
 * **操作性最優先**: PC/スマホ両対応（PWA化によりスマホでも全画面で操作可能にする）。
 * **開発速度重視**: 外部DB（Supabase等）は使わず、完全にフロントエンドのみで完結させる。
 *  MVP速度重視のため拡張性を重視して設計実装する。
